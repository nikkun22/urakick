# 裏キック団ルール — 編集ガイド

このファイル（`oki-10.mdx`）を直接編集することで、**プログラミング知識なし**で掟を改定できます。

---

## 掟を変更する方法

### GitHub web editor での手順

1. GitHub で `content/rules/oki-10.mdx` を開く
2. 右上の **鉛筆アイコン**（Edit this file）をクリック
3. 該当箇所を編集する（下記「編集ルール」参照）
4. ページ下部「Commit changes」をクリック
5. コミットメッセージを入力（例: `掟第3条テキスト変更`）
6. 「Commit directly to the main branch」を選択してコミット
7. 数分後にサイトが自動更新される

## 編集ルール

### 条文テキストの変更

```yaml
rules:
  - id: 3
    text: "ここを変更する。"   # ← text の値を書き換える
    penalty: true
    note: null
```

**注意**: `text:` のあとは必ず **ダブルクォーテーション（"）** で囲む。

### 条文の追加

既存の最後の条文の後に以下を追記（id は連番で）:

```yaml
  - id: 11
    text: "新しい掟の内容。"
    penalty: false
    note: null
```

- `penalty: true` → 罰則が発生する条文
- `penalty: false` → ルールだが罰則なし
- `note:` → 補足説明（不要なら `null`）

### 条文の削除

対象の条文ブロック（`- id:` から始まる4行）を丸ごと削除する。  
削除後は残りの条文の `id` を振り直すか、id の欠番はそのままでも可（サイトは表示順（index + 1）で番号を振るため）。

## メタデータの更新ルール（改定のたびに必ず更新）

```yaml
改定日: "2026-05-04"   # 今日の日付（YYYY-MM-DD形式）
改定回数: 1            # 前回の値 + 1
```

---

## YAML の書き方の注意点

| 正しい | 間違い | 理由 |
|--------|--------|------|
| `text: "テキスト。"` | `text: テキスト。` | クォートなしはエラーの原因に |
| インデント: 半角スペース2個 | インデント: タブ | YAMLはタブ禁止 |
| `null` | `""` / 空欄 | 空は null と書く |
